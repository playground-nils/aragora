# Stale Tasking-Issue Audit — 2026-05-14

Read-only audit of open "Open PR for ..." / "Open draft PR for ..." style **tasking issues** (auto-generated by the automation outbox publisher) that are provably satisfied by an already-merged PR.

This audit is **read-only**. No issues are closed here. Closing requires a separate explicit operator authorization scoped to exactly these 48 issues.

## Method

1. Snapshot all open issues whose title matches one of: `Open PR for *`, `Open draft PR for *`, `Open a PR for *`, `Open a draft PR for *`, `Update PR *`, `Publish existing repair *`, `Publish or confirm *`. Result: **134 open tasking issues** at snapshot time.
2. For each issue, parse the body for the `"branch"` field inside the embedded `Requested Action` JSON. Issues without a parseable branch (e.g. `Publish or confirm` reminder-style entries) are excluded from the satisfied set.
3. Build an index of merged PRs since `2026-04-20` from `gh pr list --state merged --search "merged:>2026-04-20"`. Index size: **500 PRs**.
4. For each tasking issue, look up its `branch` value in the merged-PR index keyed by `headRefName`. A match means the branch the issue requested was actually merged via some PR.
5. The merged PR's number and merge-commit SHA are recorded as evidence.

### Method limitations (deliberately conservative)

- This audit only matches by **exact branch name**. No fuzzy title matching, no body-similarity heuristics.
- It does not classify the 86 "ambiguous" issues — they remain open and untouched by this audit.
- A few of the 48 entries are "restacks" of an earlier attempt (e.g. `#6545` restacks `#6544` for the same branch); both entries are listed because both were genuinely opened, both refer to the same merged work, and both can be closed by the same evidence.
- Two entries (`#6549`, `#6657`) match merged PRs whose head ref carries a salvage/migration name that doesn't immediately resemble the tasking title; the branch-name index match is still authoritative because the body's `branch` field is the source of truth for what the publisher tried to open.

## Out of scope of this audit

- The **86 ambiguous tasking issues** that did not match a merged-PR head ref (likely classes: weekly publication reminders, restacks under renamed branches, abandoned attempts) are NOT addressed here. They require per-issue manual review and are NOT proposed for closure.
- The underlying publisher bookkeeping bug (no issue↔PR linkage maintained, so issues are never auto-closed when their corresponding PR merges) is acknowledged but not fixed here. A separate small PR is contemplated to add a `pull_request: closed` GitHub Action that closes matching tasking issues by exact branch ref.

## Evidence reproducibility

This audit can be reproduced from a clean shell. **Caveat:** the `gh` list commands below cap their input at `--limit 500` (merged PRs) and `--limit 300` (open issues). The 2026-05-14 snapshot fell within both caps (500 merged PRs since 2026-04-20, 134 open tasking issues). Re-running after the queue size has shifted, or after the 48 confirmed-stale issues have been closed, may produce a different — but smaller — set of confirmed-stale rows. The audit's evidence chain (issue body `branch` → merged PR `headRefName` → merge commit) is stable per row; only the cardinality of the set is sensitive to the cap and to subsequent closures.

The title-pattern filter below matches the same seven patterns enumerated in the Method section (`Open PR for *`, `Open draft PR for *`, `Open a PR for *`, `Open a draft PR for *`, `Update PR *`, `Publish existing repair *`, `Publish or confirm *`).

```bash
gh pr list --state merged --limit 500 --search "merged:>2026-04-20" \
  --json number,headRefName,mergedAt,mergeCommit --jq '
    [.[] | {head: .headRefName, pr: .number, merged: .mergedAt, sha: .mergeCommit.oid[0:8]}]
  ' > /tmp/merged_prs.json

gh issue list --state open --limit 300 --json number,title,body --jq '
    [.[] | select(
        (.title | startswith("Open PR for"))
        or (.title | startswith("Open draft PR for"))
        or (.title | startswith("Open a PR for"))
        or (.title | startswith("Open a draft PR for"))
        or (.title | startswith("Update PR "))
        or (.title | startswith("Publish existing repair"))
        or (.title | startswith("Publish or confirm "))
      )
     | {n: .number, branch: (.body | capture("\"branch\":\\s*\"(?<b>[^\"]+)\"") .b // null)}]
  ' > /tmp/open_tasking_issues.json

jq -r --slurpfile merged /tmp/merged_prs.json '
    .[] | . as $i |
    ($merged[0] | map(select(.head == $i.branch))) as $m |
    if ($m | length) > 0 then "\($i.n)|\($i.branch)|\($m[0].pr)|\($m[0].sha)|\($m[0].merged)" else empty end
  ' /tmp/open_tasking_issues.json | sort -t'|' -k1n
```
