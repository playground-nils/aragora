### Rescue class absorbed

**Class:** "dormant feature-flagged autonomy code surviving a rejected design".

**Absorption:** Future autonomy PRs should either (a) ship behind a flag that
is affirmatively exercised in CI, or (b) be removed from main when the design
they serve is rejected.  Leaving flag-gated code as "dormant" is an
anti-pattern because a single environment variable flip reopens the
rejected architecture without review.  The Option-B declaw pattern (keep
pure types, delete action paths, preserve future-reuse surface) is the
right pattern when some of the work is worth preserving.
