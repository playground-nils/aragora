## Publication surface

The daily-updated status page is [`docs/status/B0_BENCHMARK_TRUTH_STATUS.md`](B0_BENCHMARK_TRUTH_STATUS.md). It is regenerated by `scripts/render_benchmark_truth_status.py` and committed to a fresh `benchmark-truth-publication/<run-id>` branch each day. The Auto PR Publisher opens the draft PR if repository policy permits.

Key fields emitted:
- **Verified truth success rate** — primary metric per `TW-02`
- **No-rescue truth success rate** — direct H1 progress signal
- **Merged-only rate** — strictest settlement proof
- **In-Flight Graduation Metrics** — visibility into expected issues
- **Failure Class Distribution** — canonical buckets (`blocked_auth_failure`, `blocked_not_dispatch_bounded`, `rescue_*`, etc.)
- **Deltas** — week-over-week delta per metric

## How H1-02 composes with H1-01

Once H1-01 rev-4 (PR #6257) graduates from STAGING to canonical per the promotion criteria in `docs/benchmarks/corpus_rev4_staging.md` (landing via PR #6257), the workflow cutover will consist of pointing `scripts/measure_b0_scorecard.py` and `scripts/render_benchmark_truth_status.py` at the new canonical manifest. Until then, rev-3 remains the canonical surface and the scorecard reflects its evidence. This preserves the rev-3 honesty audit invariants and avoids conflating staged candidate metrics with canonical reality.
