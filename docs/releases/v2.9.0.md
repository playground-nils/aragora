# Aragora v2.9.0

**Target release:** 2026-04-25, pending nightly confirmation and stable `v2.9.0` tag
**Previous stable:** v2.8.0 (2026-02-16)
**Release-candidate predecessor:** v2.9.0-rc.1 (2026-04-24, sha `40a9784f4`)

## Headline

Mode 3 heterogeneous PR review is operationally real. Eight providers (Claude, GPT, Gemini, Grok, DeepSeek, Kimi, Qwen, Mistral) plus a ninth advocate-lens slot run on every shipped review path, generate signed briefs with per-finding severity, route through a calibrated triage rule that distinguishes blockers from non-blockers, and emit dissent-preserving receipts for human settlement.

This is the intended first stable release where the thesis (`docs/THESIS.md`) is operationally validated rather than aspirational. Premise 3 (heterogeneous adversarial cross-check) and premise 6 (triage routing) are both live, but final release truth is still gated on the nightly confirmation and tag.

## Upgrade notes

### From v2.8.x

This is planned as a feature release, not a breaking-change release. No mandatory migration steps are expected. After the stable tag is cut, verify your installation:

```bash
pip install --upgrade aragora-debate
python -c "import aragora; print(aragora.__version__)"
```

### Mode 3 brief consumers

If you consume `ReviewBrief.recommendation` programmatically:

- New value `approve_with_followups` may now appear (was previously only `approve_candidate`, `repair_first`, `needs_human_attention`)
- New optional field `findings_severity_counts: dict[str, int]` may appear on briefs

Both additions are backwards-compatible at the JSON schema level; legacy consumers that don't recognize them will ignore them.

### npm consumers (frontend / SDK)

If you depend on `aragora/live` or any of the npm projects (`sdk/typescript`, `examples/sveltekit`, `ide/vscode-aragora`, `docs-site`), the npm overrides in `package.json` now pin transitive dep versions to patched releases. Re-run `npm install` after pulling.
