# Frontend Routes and Feature Map

This document maps the primary UI routes in `aragora/live` to their purpose and related documentation.
It is the canonical map for the user-facing surface area.

## Canonical UI Domains

- The current Next.js app is landing-first: `/` server-redirects to `/landing/`.
- Standalone/public routes include `/landing/`, `/about`, `/pricing`, `/docs`, `/login`, and `/signup`.
- App-shell routes include `/debates`, `/inbox`, `/settings`, and the rest of the `(app)` surface.
- Hostnames such as `aragora.ai` and `live.aragora.ai` may map these surfaces differently per deployment. Verify host-level routing in deployment config instead of assuming it from this route tree alone.

Local development uses `http://localhost:3000` with `NEXT_PUBLIC_API_URL` and `NEXT_PUBLIC_WS_URL` from `.env.local`.
